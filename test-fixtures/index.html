Detector A — verify on real sites (not here)

Detector A wraps the page’s own fetch/XHR, and cross-origin requests from this local page are blocked by CORS before the wrap sees a usable response. So Detector A must be exercised on the real sites whose same-origin export calls it is designed to catch:

  • Komoot — open a tour, click Export GPX. Expect [detector:A] … source=fetch after the response completes (POST → bytes, no .gpx in the URL).
  • Strava — an activity’s Export GPX (/activities/<id>/export_gpx, no extension in the path — exercises content-type / content-disposition sniffing).
  • Garmin Connect / AllTrails / Ride with GPS — any in-page GPX/TCX/KML export button.
  • At least one site that exports via XMLHttpRequest rather than fetch → expect source=xhr.
