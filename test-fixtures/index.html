Color The Map Importer — test fixtures

Load the unpacked extension, sign in to Color The Map (toolbar icon → Connect) with at least one map, then work through the sections below. Serve this page over HTTP — e.g. npx http-server test-fixtures.

The sample files in test-fixtures/files/ are committed to the repo. valid.gpx is a real ride that starts and ends in a public area (no sensitive location data); the other tracks are public example files.

Each sample offers up to three triggers:

  • link — a normal anchor (Detector C). Clicking it lets the browser download as usual and opens the Send-to-CTM dialog. Detector C flags by file extension only — it never reads the bytes.
  • fetch / XHR — fires the page's own request for the file (Detector A), which intercepts the response and reads the bytes. “+ save” also writes the file to disk, mimicking a real “Export” button (the extension neither causes nor blocks that save).

Valid samples — should detect and import

Expect: every trigger opens the dialog. Pick a map (the selector shows only when you have more than one) and Send → the track appears in that Color The Map within a few seconds.

    Client-side rejects — content fails the extension's sniff

    Expect: the fetch / XHR buttons open no dialog — Detector A reads the bytes and rejects them on content. The link still opens a dialog (Detector C flags by extension only); on Send, Color The Map rejects it and the dialog shows CTM's actual error.

      Server-side rejects — valid format, nothing to import

      Expect: every trigger opens the dialog (the content passes detection). On Send, Color The Map accepts the upload but finds no track, so it returns an error the dialog surfaces verbatim.

        Detector C — dynamically added link (MutationObserver)

        The inserted link should be wired the moment it appears — clicking it opens the dialog for valid.gpx.

        Detector A — content-sniff edge cases (negatives/)

        Extra bodies that look like GPS by URL but aren't. These exercise Detector A's structural sniff specifically; the fetch / XHR buttons must open no dialog.

          Detector A — real-site sanity check (optional)

          The local buttons cover the mechanism. For real-world shapes the static server can't reproduce (POST → bytes, custom content-types), spot-check a real export: Komoot (open a tour → Export GPX) or Strava (an activity's Export GPX). Each should open the dialog with the intercepted bytes.
